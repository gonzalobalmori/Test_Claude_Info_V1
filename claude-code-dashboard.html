Qué es Claude

El punto de entrada: qué ofrece Claude, cómo obtener acceso y cómo configurarlo desde el primer día.

Tres modos de interacción

Claude Chat Disponible
Conversación avanzada en claude.ai o en la app desktop. Redacción, análisis, síntesis de documentos, brainstorming. Accesible desde el navegador o desde Claude Desktop.
Claude Code Disponible
Agente de código integrado en Claude Desktop. Genera, edita y ejecuta código en tu máquina, crea archivos, automatiza tareas y trabaja con tu sistema de ficheros.
Claude Cowork No disponible en Accenture
Colaboración en tiempo real con múltiples usuarios sobre el mismo contexto. No está habilitado en las licencias corporativas de Accenture por ahora.

Cómo empezar

01
Solicitar licencia a Accenture
Accede al portal interno de herramientas y solicita acceso a Claude. La aprobación suele tardar 1-2 días hábiles. Solicitar licencia
02
Descargar Claude Desktop
Descarga e instala la aplicación de escritorio para Windows. Necesaria para usar Claude Code y los schedules. Descargar Claude Desktop
03
Instalar Git
Necesario para control de versiones. Permite deshacer cambios y mantener un historial de lo que Claude genera. Descargar Git
04
Primer acceso
Abre Claude Desktop e inicia sesión con tu cuenta Accenture (usuario@accenture.com). Se sincronizará automáticamente con tu licencia corporativa.

Una vez dentro

Activar capabilities
En Settings > Capabilities, activa:
  • Memory — Claude recuerda contexto entre conversaciones
  • Visuals — Claude puede analizar imágenes y generar gráficos
  • Code execution & file creation — Claude ejecuta código y crea archivos en tu máquina
Controlar el consumo (Usage)
En Settings > Usage puedes ver cuántos tokens has consumido en el mes. De momento existe un límite mensual de uso.

Claude usa modelos más ligeros para tareas sencillas. Sonnet es el modelo recomendado. Opus es el más potente, pero consume más cuota.
Migrar memoria desde otro LLM
Si venías usando ChatGPT u otro asistente, puedes exportar tu perfil en 2 pasos:

Paso 1: Copia el prompt de abajo y pégalo en tu ChatGPT (u otro LLM). Copia la respuesta que te devuelva.

Paso 2: En Claude, ve a Settings > General y pega lo que te haya devuelto el otro LLM en el campo "What personal preferences should Claude consider in responses?"
Please generate a comprehensive memory export for me to use with a new AI assistant. Include: 1. My professional profile: role, industry, main responsibilities 2. My communication preferences: tone, level of detail, format I prefer 3. Key projects or contexts I frequently discuss 4. My technical expertise level in different areas 5. Recurring tasks or workflows I use you for 6. Any specific instructions or preferences I have told you about 7. Important background knowledge about my work or goals Format this as a structured profile that another AI assistant can use to understand me quickly.
Cambiar el estilo de respuesta
En Profile > Response style puedes personalizar cómo responde Claude:
  • Concise — respuestas directas y cortas
  • Explanatory — más detalle y contexto
  • Formal — tono profesional
  • Learning — explicaciones didácticas para aprender
  • Custom — define tu propio estilo con texto libre

02

Claude Code

Cómo trabajar de forma efectiva con el agente de código: modos de trabajo, CLAUDE.md e integración con Git.

Modos de trabajo

Ask permissions
Claude te pide confirmación antes de cada acción. Control total, pero más lento.

Ideal para: aprender cómo funciona Claude o tareas delicadas.
Auto accept edits
Claude ejecuta acciones automáticamente sin pedir permiso. Muy rápido, pero puede hacer cambios no deseados si el contexto no está bien definido.

Ideal para: tareas repetitivas bien definidas donde confías en el resultado.
Plan mode Recomendado
Claude primero propone un plan detallado y espera tu aprobación antes de actuar. Equilibrio ideal entre control y velocidad.

Ideal para: casi todo. Revisa la estrategia antes de ejecutar.

CLAUDE.md — tu manual de instrucciones para Claude

¿Qué es el CLAUDE.md?

Es un archivo de texto en formato Markdown que Claude lee automáticamente al inicio de cada sesión. Es tu forma de darle instrucciones permanentes sin tener que repetirlas cada vez.
CLAUDE.md global
Claude lo lee en todas tus sesiones de Claude Code, independientemente del proyecto. Ideal para instrucciones de estilo, tono y preferencias generales.

Ubicación: C:\Users\(tu_id)\.claude\CLAUDE.md

Cómo crearlo: escribe /init en Claude Code, o pide: "Crea mi CLAUDE.md global con estas instrucciones…"
CLAUDE.md de proyecto
Claude lo lee solo en sesiones dentro de ese proyecto. Tiene prioridad sobre el global — si hay conflicto, prevalece el de proyecto.

C:\ruta\proyecto\CLAUDE.md

Cómo crearlo: abre la carpeta en Claude Code y pide: "Crea un CLAUDE.md para este proyecto con…"
Estructura modular recomendada

Claude lee mejor archivos de menos de 200 líneas. Para proyectos complejos, usa estructura modular:
  • CLAUDE.md principal con punteros a otros archivos
  • contexto.md — quién eres, tu equipo, el negocio
  • estilo.md — cómo escribes, nombras variables, estructuras el código
  • flujo.md — tu proceso de trabajo, commits, tests…
Buena práctica: registrar correcciones

Cuando corrijas a Claude, pídele que lo anote:

"Anota esto en mi CLAUDE.md global para que no vuelvas a cometer el mismo error."

Así construyes un documento vivo que mejora con cada interacción.

Organización por proyectos y control de versiones

Selecciona tu carpeta de trabajo
Al abrir Claude Desktop, selecciona la carpeta del proyecto. Todo lo que Claude genere se creará dentro de esa carpeta.

No mezcles proyectos distintos en la misma carpeta — Claude mezclaría contextos. Crea un CLAUDE.md en cada proyecto con su contexto específico.
Control de versiones con Git — desde Claude Desktop
No necesitas usar la terminal. Pídele a Claude directamente en lenguaje natural:
  • "Haz un commit de seguridad antes de empezar." — Claude inicializa git si hace falta y crea el punto de restauración.
  • "Muéstrame el diff de los últimos cambios." — Claude te muestra exactamente qué ha modificado.
  • "Deshaz los últimos cambios." — Claude ejecuta el rollback automáticamente.

Schedules

Automatiza tareas recurrentes con lenguaje natural. Claude ejecuta flujos en tu máquina sin intervención.

Qué son y cómo crearlos

Qué es un schedule

Una tarea que le describes a Claude en lenguaje natural, con una frecuencia (cada día, cada lunes, cada hora...) y él la ejecuta automáticamente sin que tengas que intervenir.

Para crearlos: en Claude Desktop, escribe directamente:
"Crea un schedule que haga X cada Y." Claude configurará la tarea por ti.
  Requisito técnico

Para que cualquier schedule funcione:
  • El PC debe estar encendido en el momento de ejecución
  • Claude Desktop debe estar abierto (puede estar minimizado)
  • Los schedules corren en local — no funcionan desde la nube

Ejemplo 1: curación de contenido IA para LinkedIn

PASO 1
Revisa tu Outlook
Cada lunes por la mañana accede a tu carpeta de newsletters de IA
PASO 2
Extrae novedades clave
Identifica los 3-5 temas más relevantes en IA generativa
PASO 3
Genera borrador LinkedIn
Redacta un post con tu voz, formato y hashtags habituales
PASO 4
Te lo envía por mail
Recibes el borrador en tu bandeja para revisar y publicar
Nota: La subida directa a LinkedIn no está disponible por ahora — Accenture no tiene ningún MCP (Model Context Protocol) activo de momento.

Ejemplo 2: dashboard auto-actualizado

Condición previa
Debes tener el dashboard ya creado. El dashboard debe leer datos de un archivo Excel, CSV, base de datos o API accesible desde tu máquina. Claude no puede acceder a datos que no estén en tu máquina local.
Cómo pedirlo a Claude
"Crea un schedule que cada viernes a las 17:00 lea el archivo ventas.xlsx de mi carpeta Documentos, actualice el dashboard.html con los datos de esa semana y guarde el resultado."

Claude configurará la tarea automáticamente.
01
Claude lee los datos
Accede al archivo en la ruta configurada
02
Regenera el HTML del dashboard
Actualiza gráficos, tablas y KPIs con los nuevos datos
03
Guarda el archivo actualizado
El lunes ya tienes el dashboard al día listo para compartir

Skills

Extiende las capacidades de Claude con instrucciones especializadas reutilizables en todos tus proyectos.

Qué son las skills

Una skill es un conjunto de archivos .md con instrucciones detalladas para que Claude domine una tarea concreta: aplicar brand guidelines, generar slides con un formato específico, redactar en el tono de tu empresa, analizar datos con tu metodología...

Se almacenan en: C:\Users\(tu_id)\.claude\skills\
Formato y buenas prácticas
  • Son archivos markdown (.md) — legibles por Claude y por humanos
  • Menos de 200 líneas por archivo — estructura modular para skills complejas
  • Un archivo principal SKILL.md que orquesta a los demás
  • Archivos especializados: brand.md, rules.md, examples.md...
  • Pide a Claude que registre correcciones en la skill para que aprenda
Cómo crear una skill desde cero
No necesitas escribir la skill manualmente. Dile a Claude:

"Crea una skill llamada [nombre] que sepa [descripción detallada de lo que debe hacer]. Usa estructura modular con un archivo principal y archivos especializados."

Desde Claude Chat: Claude crea los archivos en la ubicación actual. Después debes mover la carpeta a C:\Users\(tu_id)\.claude\skills\ para que esté disponible en todos tus proyectos.

Desde Claude Code: Claude crea la skill directamente en la carpeta de skills — no hace falta moverla.

Instalar una skill descargada

1
Descargar los archivos
Descarga los archivos de la skill
→
2
Crear la carpeta skills
Si no existe, créala en: C:\Users\(tu_id)\.claude\skills\
→
3
Crear subcarpeta con nombre exacto
El nombre debe coincidir exactamente con el nombre con el que llamarás a la skill. Ej: accenture-brand
→
4
Pegar los archivos
Copia todos los archivos descargados dentro de la subcarpeta. Lista para usar.

Cómo usar una skill

Llamarla explícitamente
Escribe /nombre-skill en Claude Code para invocarla directamente. Claude cargará todas las instrucciones antes de responder.

Ejemplo: /accenture-brand
Detección automática
No siempre es necesario llamarla. Si el contexto de tu solicitud encaja con una skill instalada, Claude la activará automáticamente.

  Las skills no se actualizan solas
Si alguien del equipo mejora una skill, tu copia local no cambia. Debes volver a descargarla y reemplazar los archivos.

Repositorio de skills

Cómo el equipo comparte, versiona y mantiene un repositorio común de skills para evitar trabajo duplicado.

¿Por qué necesitamos un repositorio compartido?

Trabajo duplicado
Distintos miembros del equipo crean skills similares de forma independiente, sin saber que ya existen versiones mejores.
Versiones desincronizadas
Cada uno tiene una versión distinta de la misma skill. Mejoras y correcciones no se comparten con el equipo.
Sin historial de cambios
No hay forma de saber quién cambió qué ni cuándo, ni de volver a una versión anterior que funcionaba mejor.

Cómo compartir skills hoy

Carpeta compartida en SharePoint Sugerencia temporal
Mientras no exista un repositorio oficial, puedes usar una carpeta compartida en SharePoint como solución provisional.

Cada persona sube su carpeta de skill y el equipo accede a ella desde ahí.
skills/ accenture-brand/ # brand Accenture linkedin-posts/ # posts LinkedIn report-generator/ # informes ...
Cómo saber si hay versión más reciente

SharePoint muestra la fecha de modificación de cada archivo. Si alguien actualizó una skill después de que tú la descargaste, la fecha te lo indicará.

Para actualizar: descarga de nuevo la carpeta y reemplaza los archivos en tu .claude/skills/ local.
  Requiere mantenimiento del equipo

Cualquier repositorio centralizado necesita un responsable que valide las skills antes de publicarlas. Sin un owner claro, el repositorio se degrada rápidamente.
